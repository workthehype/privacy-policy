Important: This Privacy Policy applies to the Location Attendance mobile application ("App"). By using our App, you agree to the collection and use of information in accordance with this policy.

Location Attendance ("we", "our", "us") respects your privacy and is committed to protecting your personal data. This Privacy Policy explains how we collect, use, store, and safeguard information when you use our mobile application.

1. Information We Collect

1.1 Personal Information

We collect the following personal information:

  • Name: To identify users within the organization
  • Email Address: For account creation and communication
  • Phone Number: Optional, for contact purposes
  • Employee/User ID: For organizational identification
  • Organization Details: Company name and organization code

1.2 Location Data

Background Location Collection:

Location Attendance collects location data, including background location, to enable the following features:

  • Attendance tracking and verification
  • Geo-fencing validation (ensuring users are at designated work locations)
  • Workplace presence verification
  • Preventing fraudulent or proxy attendance

Location data may be collected even when the app is closed or not in use. However, this collection only occurs:

  • After you provide explicit consent through our in-app disclosure
  • When you are actively checked in for work
  • To verify your presence at the assigned work location

1.3 Device Information

We collect device-related information for security and functionality:

  • Device ID: To prevent multiple device logins and ensure security
  • App Version: To track which version users are using
  • Operating System: For compatibility and support

1.4 Attendance Data

  • Check-in and check-out timestamps
  • Break session durations
  • GPS coordinates at check-in/check-out
  • Working hours and attendance history

1.5 Leave Requests

  • Leave application details (dates, reason)
  • Leave approval/rejection status

2. How We Use Your Data

2.1 Primary Purposes

Your data is used strictly for:

  • Attendance Management: Recording and tracking employee attendance
  • Location Verification: Ensuring users are at designated work locations
  • Fraud Prevention: Preventing fake, proxy, or fraudulent attendance
  • Report Generation: Creating attendance reports for authorized administrators
  • Leave Management: Processing and tracking leave requests
  • Notifications: Sending attendance-related notifications to admins
  • Account Security: Device locking to prevent unauthorized access

2.2 Data Sharing

We do NOT sell, rent, or share your personal data with third parties for marketing purposes.

Your data is only accessible to:

  • You: Your own attendance and profile data
  • Your Organization's Admins: Authorized company administrators can view attendance data of users in their organization
  • Super Admins: System administrators for app management purposes

2.3 Multi-Tenancy

Our app uses a multi-tenancy architecture, which means:

  • Each organization's data is isolated and separate
  • Company admins can only see data from their own organization
  • Data is never shared between different organizations

3. Location Data - Detailed Disclosure

3.1 Why We Collect Location Data

Feature Location Type Purpose
Check-In Foreground (Precise) Verify user is at assigned work location
Check-Out Foreground (Precise) Verify user is still at work location when leaving
Periodic Verification Background (Approximate) Ensure user remains at work location during shift
Distance Calculation Foreground (Precise) Show distance from work location in real-time

3.2 When Location is Collected

  • During Check-In: When you tap "Check In" button
  • During Check-Out: When you tap "Check Out" button
  • While Working: Periodically to verify presence (only when checked in)
  • NOT Collected: When you are checked out or not working

3.3 User Control Over Location

You have full control:

  • Location permissions are requested with a clear explanation
  • You can grant or revoke location access anytime via device settings
  • You can choose "Allow only while using the app" or "Allow all the time"
  • Disabling location will prevent attendance features from working

4. Data Storage and Security

4.1 Where Data is Stored

  • Cloud Storage: Google Firebase Firestore (secure cloud database)
  • Location: Data centers in the United States
  • Encryption: All data is encrypted in transit (HTTPS) and at rest

4.2 Security Measures

We implement the following security measures:

  • Authentication: Firebase Authentication for secure login
  • Access Control: Role-based permissions (User, Company Admin, Super Admin)
  • Device Locking: One device per user to prevent unauthorized access
  • Data Isolation: Organization-based data separation
  • Firestore Security Rules: Server-side validation of all data access

4.3 Data Retention

  • Attendance Records: Retained indefinitely for historical records
  • User Accounts: Retained while account is active
  • Deleted Accounts: Data is permanently deleted within 30 days of account deletion
